# 0.8.1

## New features

### Definition lists
- PHP Markdown Extra / Pandoc definition list syntax:
  ```
  Term
  : Definition one
  : Definition two
  ```
- New AST nodes: `OrcaBlock.DefinitionList`, `OrcaDefinitionListItem`
- Pre-processing extraction layer (same pattern as footnotes/front matter)
- Multiple definitions per term, multiple terms per list
- Inline formatting supported in terms and definitions
- Compose renderer with indented definitions under bold terms
- Configurable via `OrcaDefinitionListStyle` (term style, indentation, spacing)
- Light and dark theme presets

### Abbreviations
- PHP Markdown Extra abbreviation syntax:
  ```
  *[HTML]: Hyper Text Markup Language
  *[CSS]: Cascading Style Sheets
  ```
- New AST node: `OrcaInline.Abbreviation` (text + title)
- Definitions are extracted and removed from output
- Whole-word matching replaces occurrences in inline content (no partial words)
- Recursive replacement through bold, italic, strikethrough, links, etc.
- Compose renderer with underline + subtle background style
- Configurable via `OrcaInlineStyle.abbreviation`
- Light and dark theme presets

## Style changes

- Added `OrcaDefinitionListStyle` data class (termStyle, definitionIndent, termSpacing, definitionSpacing)
- Added `definitionList` field to `OrcaStyle`
- Added `abbreviation` span style to `OrcaInlineStyle`
- Updated `OrcaDefaults.darkStyle()` with dark theme definition list and abbreviation styles

## Breaking changes

- `OrcaBlock` sealed interface has a new variant (`DefinitionList`) -- exhaustive `when` expressions on `OrcaBlock` will fail to compile until the new branch is added
- `OrcaInline` sealed interface has a new variant (`Abbreviation`) -- exhaustive `when` expressions on `OrcaInline` will fail to compile until the new branch is added

## Tests

- Definition list parsing: simple, multiple definitions, multiple terms, inline formatting in terms, mixed with regular content
- Abbreviation parsing: extraction and application, multiple abbreviations, word boundary matching, abbreviations inside bold, definition removal
- Reference-style link tests: full reference, short reference, reference images, multiple definitions (confirming existing support)
